Git 과 GitHub

깃, 깃허브는 기본적으로 어떠한 파일, 또는 디렉토리의 버전을 관리하기 위해서 사용한다.

Git
깃은 내가 버전별로 관리할 파일을 내 컴퓨터 ( 로컬저장소 ) 에 두고 관리하는 것을 말한다.

GitHub
깃허브는 내가 버전별로 관리할 파일을 깃허브 서버 ( 원격저장소 ) 에 두고 관리하는 것을 말한다.

깃과 깃허브의 가장 큰 차이점은 저장소의 위치라고 생각한다.

원격저장소에 있는 파일들은 해당 저장소에 접근권한이 있는 모든 사람들이 파일을 다운로드 받고 수정하여 업데이트가 가능하다.
이러한 방면에서 엄청나게 우수하기 때문에 협업을위해 사용된다고 한다.

-------------------------------------------------------------------------------------------


개념

깃과 깃허브는 기본적으로 내 로컬저장소를 기본으로 사용한다.


Git

우선 깃에게 버전을 관리할 폴더를 알려줘야한다.

[폴더1] 의 버전을 관리하기 위해 우리는 터미널에서 [폴더1] 의 위치로 디렉토리를 변경한다.

git init 명령어를 사용하여 깃에게 [폴더1] 을 버전관리 하겠다고 알려준다. ( 숨겨진 폴더 git 이 생성됨 )

이제 [폴더1] 의 내용 ( 파일 ) 들이 변경될 때 깃이 변경사항을 기억하고 있는다.

변경 사항을 확인하거나 상태를 확인하려면 git status 명령어를 사용한다. 텍스트 파일을 만들고 git status를 사용해보자.

이렇게 변경된 사항을 새로운 버전을 만들면서 저장하기 위해서는 git commit 이라는 명령어를 사용해야한다.

새로운 버전을 생성하기에 앞서 우리는 깃에게 어떠한 파일을 새로운 버전에 적용할것인지 알려줘야한다.

[폴더1] 안에 [파일1,파일2] 가 있었고 둘 다 수정했지만 [파일1] 의 변경 사항만 저장하고 싶을 수 있기 때문이다.

깃에게 어떠한 파일을 적용할 것인지 알려주는 방법은 add 명령어를 사용하면된다.

git add [파일명] 이런식으로 알려준다.

이렇게 깃에게 commit 할 파일을 알려줬다면 git commit -m "메모내용" 명령어를 사용하여 commit 를 해보자.

변경사항이 적용된 새로운 버전이 만들어지면서 과거와 현재를 버전으로 관리할 수 있게된다.



GitHub

깃허브는 Repository ( 이하 레포지토리 ) 라는 개념을 사용하는데 내 컴퓨터의 폴더와 같은 개념이다. 

버전 관리할 폴더에서 git init 를 사용한다.

git remote add origin 레포지토리 주소 를 사용하여 내가 버전관리하던 로컬저장소 폴더와 원격저장소 레포지토리를 연결한다.

내 로컬 저장소 사용법과 동일하게 사용후 commit 까지 완료했고 해당 변경사항을 원격저장소 레포지토리에 적용하고 싶다면,

git push origin master 명령어를 사용하여 업로드 할 수 있다.

이 부분에서 깃허브 아이디, 비밀번호를 입력하게 되는데 보안이 낮은 비밀번호는 push, pull 등 원격 관련된 명령어는 차단당한다.

해결 방법으로는 권한이 부여된 토큰을 발급받아 비밀번호 대신 사용할 수 있다.

이제 원격저장소에 업로드 하는것까지 사용이 가능해졌다. 반대로 다운로드 받을때는 git pull 을 사용하면 된다.


-------------------------------------------------------------------------------------------
git pull 과 git clone 의 차이점

git clone 저장소주소

git init + git remote add origin 저장소주소 + git pull origin master

위 두 코드는 완벽하게 동일한 기능을 한다.
-------------------------------------------------------------------------------------------


정리

git init ( 버전 관리 할 폴더를 지정함 )

코드 작성 및 수정 ( 변경사항 )

git add ( commit 할 파일을 git 에게 알려줌 )

git commit ( 지정된 파일을 업데이트 하면서 새로운 버전으로 만듦 )

git remote add origin 저장소주소 ( init 된 로컬저장소와 원격저장소 연결 )

git push origin master ( 현재까지의 history 를 원격저장소에 반영 )

git pull origin master ( init 된 폴더에 remote 된 원격저장소 ( 레포지토리 ) 의 history 를 받아온다 )

git clone 명령어는 내가 위치한 디렉토리에 init + remote + pull 까지 한꺼번에 해준다.
